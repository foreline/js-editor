# Editor architecture review

This document outlines the tasks for overviewing the architecture of the editor, including core components, block types, and their interactions. The goal is to provide a clear and concise reference for developers working on the project, ensuring consistency and maintainability across the codebase.

## Problem Statement
We are having troubles to release the editor because of various small bugs, related to cursor position, block transformations, and markdown conversions and so on. We need to get a better understanding of the editor's architecture to identify the root causes of these issues and implement effective solutions.
The timeouts and debouncing in the event system are also causing some unexpected behaviors.
We are having troubles with text automatic transformations (e.g., typing `- ` to create a list; typing ``inline code`` to create inline code; typing ``` to create a code block and so on) and the toolbar button states not updating correctly after block transformations.
