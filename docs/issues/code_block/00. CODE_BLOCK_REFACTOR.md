# Code Block issues and refactoring proposal

This document outlines a proposal for refactoring or rewriting the Code Block implementation in the editor. The goal is to address existing issues, improve maintainability, and enhance the user experience when working with code blocks.

The code block is a very important feature for our users, especially those who use the editor for technical writing, documentation, or coding-related content. However, we have identified several issues with the current implementation, including:
- Inconsistent behavior when applying transformations (e.g., markdown triggers, toolbar buttons).
- Problems with cursor position and block transformations leading to unexpected results.
- Users are unable to remove the code block by pressing backspace at the beginning of the block, even if the block is empty.
- Code block created by typing triple backticks has some text content inside it, which is not expected (e.g. `Plain textJavaScriptTypeScriptPythonJavaC#PHPCSSHTMLJSONSQLBash`).
- Code highlighting is not working after the code block is created by typing triple backticks.
- The toolbar button for code block is not updating its state correctly after transformations.
- The toolbar button click does not toggle the code block state as expected.
