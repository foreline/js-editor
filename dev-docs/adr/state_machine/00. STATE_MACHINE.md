# ADR-00: Replace Boolean Flags with Editor State Machine

## Status
Accepted

## Context
- The editor relies on two boolean flags — `isCreatingBlock` and `isConvertingBlock` — to prevent race conditions during block creation and conversion. These flags guard the `ensureDefaultBlock()` check in both the input event handler and `_performUpdate()`.
- The flags are editor-wide, not per-operation. If two rapid operations overlap (e.g., user types `# ` and immediately hits Enter), only one flag can be active. The `requestAnimationFrame`/`setTimeout` clearing is timing-dependent.
- `update()` is called from many places (input handler, paste handler, KeyHandler, `Toolbar.after()`, `convertBlockType()`, `addDefaultBlock()`). The 50ms debounce coalesces these into a single `_performUpdate()`, but the `ensureDefaultBlock()` guard only runs once per batch, and may see stale flag state.
- Four separate issues in ISSUES.md trace back to flag-timing race conditions, indicating the root cause is the flag-based approach itself.

## Decision

Replace the two ad-hoc boolean flags with an `EditorStateMachine` class that enforces explicit state transitions and provides an operation queue for requests arriving during non-IDLE states.

### Design

**New class:** `src/utils/EditorStateMachine.js`

States:
```
IDLE ──► CREATING ──► IDLE
IDLE ──► CONVERTING ──► IDLE
```

Methods:
| Method | Description |
|--------|-------------|
| `startCreating()` | Transition IDLE → CREATING; returns false if not IDLE |
| `finishCreating()` | Transition CREATING → IDLE; flushes queued operations |
| `startConverting()` | Transition IDLE → CONVERTING; returns false if not IDLE |
| `finishConverting()` | Transition CONVERTING → IDLE; flushes queued operations |
| `isBusy()` | Returns true when not IDLE |
| `enqueue(fn)` | Executes immediately if IDLE; defers if busy |
| `reset()` | Error recovery: force IDLE, clear queue |

Properties:
| Property | Description |
|----------|-------------|
| `state` | Current state string (IDLE, CREATING, CONVERTING) |
| `isCreating` | Boolean, true when state is CREATING |
| `isConverting` | Boolean, true when state is CONVERTING |

**Integration in `Editor.js`:**
- Single `_stateMachine` instance created in the constructor
- Backward-compatible getters/setters for `isCreatingBlock` and `isConvertingBlock` that delegate to the state machine
- Guard checks in input handler and `_performUpdate()` use `_stateMachine.isBusy()`
- `addDefaultBlock()`, `convertBlockType()`, and `createNewBlock()` use explicit transitions
- Queue allows operations that arrive during transitions to execute after returning to IDLE

**Affected components:** `Editor.js` only — blocks, toolbar, and parser are unchanged.

## Alternatives Considered

| Alternative | Pros | Cons | Why rejected |
|-------------|------|------|--------------|
| Keep boolean flags | No changes needed | Timing-dependent, caused 4+ bugs, no queue | Root cause of recurring race conditions |
| Full transaction system | Atomic batched operations, rollback | Heavy refactor, much larger scope | Over-engineering for current needs; state machine is a stepping stone |
| Mutex / lock pattern | Prevents concurrent access | No queue semantics, harder to debug | State machine is more explicit and self-documenting |

## Consequences

### Positive
- State transitions are explicit and logged, making race conditions easier to diagnose
- Invalid transitions (e.g., starting a conversion while already creating) are guarded and logged rather than silently conflicting
- Operation queue prevents dropped operations when events overlap
- Backward-compatible: existing code reading `isCreatingBlock`/`isConvertingBlock` continues to work via getters

### Negative
- Adds one new class and import
- Slightly more indirection when debugging flag state

### Neutral
- No change to the update cycle debouncing — the 50ms debounce remains unchanged
- `requestAnimationFrame` and `setTimeout` clearing patterns remain in `addDefaultBlock()` and `createNewBlock()`; they now call `finishCreating()` instead of setting a flag
