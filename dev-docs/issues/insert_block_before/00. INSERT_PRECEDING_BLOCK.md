# Insert Preceding Block issue

This document outlines an issue/proposal related to inserting a new block before the current block in the editor. The goal is to allow users to easily insert a new block above the current block, improving the editing experience and flexibility when working with content.

## Current behavior
Currently users are unable to insert a new preceding block. They can only insert a new block below the current block by pressing Enter at the end of the block.

## Proposed solution
To enable inserting a new block before the current block, we can implement the following changes:
- Automatically insert a new block above the current block when the user presses Enter at the beginning of the block.

## Scenario
- User created one block of any type (e.g., paragraph, heading).
- User wants to add a new block before the current block.
