# Markdown Replacements issues

Thid document outlines an issue/proposal related to markdown replacements in the editor.

## Current state
Currently, the editor supports markdown replacements for certain patterns (e.g., typing `- ` or `* ` to create a list, typing hash symbols `#` to create headings, typing triple backticks to create code blocks). However, we are missing some expected behaviors and have identified several issues with the current implementation, including:
- Typing double asterisks `**` does not create bold text as expected.
- Typing single asterisk `*` does not create italic text as expected.
- Typing double underscores `__` does not create bold text as expected.
- Typing single underscore `_` does not create italic text as expected.
- Typing text in single backticks does not create inline code as expected.

## Goals
We want to ensure that the markdown replacements work as expected for all common markdown patterns, providing a smooth and intuitive editing experience for users who rely on markdown syntax.
